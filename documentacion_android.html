RecyclerView para Listas Eficientes

El RecyclerView es el componente esencial para mostrar listas de datos grandes de manera eficiente. Reutiliza las vistas de los elementos que salen de la pantalla para mostrar los nuevos que entran, evitando la creación innecesaria de objetos y mejorando drásticamente el rendimiento.

Componentes Clave

  • Adapter: Conecta la fuente de datos (una lista de objetos) con el RecyclerView.
  • ViewHolder: Mantiene las referencias a las vistas de un solo ítem (ej. un TextView, un ImageView).
  • LayoutManager: Decide cómo se posicionan los ítems (lineal, en cuadrícula, etc.).

Room Database para Persistencia Local

Room es una capa de abstracción sobre SQLite que simplifica enormemente el trabajo con bases de datos locales en Android. Ofrece seguridad en tiempo de compilación para las consultas SQL y reduce la cantidad de código repetitivo.

Componentes Clave de Room

  • Entity (@Entity): Una clase que representa una tabla en la base de datos.
  • DAO (@Dao): Una interfaz que define los métodos para acceder a la base de datos (consultar, insertar, etc.).
  • Database (@Database): La clase principal que conecta las entidades y los DAOs, y construye la base de datos.

Ejemplo 2: Lista de Tareas (Más Sencillo y Entendible)

Para entender el concepto de Room de la forma más simple posible, imaginemos una aplicación de "Lista de Tareas". Solo necesitamos una tabla para guardar las tareas.

Paso A: La Entidad `Tarea`

Definimos cómo se verá una tarea en la base de datos. Cada tarea tendrá un ID único, un título y un booleano para saber si está completada.

@Entity(tableName = "tareas")
public class Tarea {
    @PrimaryKey(autoGenerate = true)
    public int id;
    
    public String titulo;
    
    public boolean completada;
}

Paso B: El DAO `TareaDao`

Definimos las acciones que podemos realizar: obtener todas las tareas, insertar una nueva y eliminar una.

@Dao
public interface TareaDao {
    @Query("SELECT * FROM tareas")
    List getAll();
    
    @Insert
    void insert(Tarea tarea);
    
    @Delete
    void delete(Tarea tarea);
}

Paso C: La Base de Datos `MiBaseDeDatos`

Juntamos todo. Esta clase le dice a Room que nuestra base de datos contendrá una tabla de `tareas` y que usará `TareaDao` para acceder a ella.

@Database(entities = {Tarea.class}, version = 1)
public abstract class MiBaseDeDatos extends RoomDatabase {
    public abstract TareaDao tareaDao();
}

Con estos tres componentes, ya tienes una base de datos local completamente funcional y segura para gestionar una lista de tareas.
